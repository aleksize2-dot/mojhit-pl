1. Обзор системы (High-Level)
Приложение представляет собой чат-платформу, где пользователь взаимодействует с различными AI-агентами (продюсерами) для генерации музыкальных промптов. Система имеет три уровня монетизации: подписки (уровни доступа к LLM), внутренняя валюта (Монеты/Ноты) и временные пакеты (эксклюзивные агенты).

2. Модель данных (Database Schema)
User Profile & Wallet
user_id: UUID

coins_balance: (Hard currency) — покупные, не сгорают.

notes_balance: (Soft currency) — бонусные, за активность.

subscription_tier: Enum (FREE, BASIC, PRO).

llm_model_access: Enum (GEMINI_ONLY, ALL_MODELS).

Producers (Agents)
producer_id: String (e.g., 'marek', 'maja', 'kosa', 'basia').

is_exclusive: Boolean (доступен всем или только по пакету).

system_prompt: Text (уникальный для каждого).

config: JSON (аватар, цвет чата, приветствие, музыкальные теги).

Permissions & Access (Crucial)
user_producers: Таблица связей (User <-> Producer).

expiry_date: Timestamp (null для базовых, дата для пакетных).

Action: 1.  В таблицу user_producers добавляется запись с новым producer_id.
2.  Устанавливается expiry_date (например, +7 или +30 дней).

Frontend Effect: В карусели выбора агентов появляется новая иконка. По истечении времени иконка становится неактивной (Gray scale) или исчезает.

5. API Endpoints
POST /api/chat/message:

Принимает: user_id, producer_id, message.

Логика: Проверяет права доступа -> Выбирает System Prompt агента -> Дергает LLM -> Возвращает ответ.

POST /api/chat/generate-prompt:

Парсит ответ LLM между тегами ---PROMPT--- и ---END---.

Списывает валюту (Монеты/Ноты).

Отправляет промпт в музыкальный движок (kie.ai).

GET /api/producers/list:

Возвращает список доступных пользователю агентов с их конфигами.

6. UI/UX требования (для Frontend)
Agent Switcher: Горизонтальный список аватаров (крупные Close-up иконки).

Theme Switcher: При смене агента меняется CSS-переменная --accent-color (Синий для Марека, Розовый для Майи, Кислотный для Косы).

Wallet Display: Четкое разделение 🟡 (Монеты) и 🎵 (Ноты).

Typing Indicator: Индивидуальный текст для каждого (напр. "Maja układa rymy...").

7. Обработка валюты
Priority: Система всегда сначала списывает Ноты, затем Монеты.

Validation: Если баланс < 1 песни — блокировка кнопки "Generuj hit" с оффером пополнения.

Комментарий для разработчика:
Бизнес-логика заточена под польский рынок (BLIK). Агенты должны восприниматься как живые личности. Особое внимание уделить парсингу промптов, чтобы в музыкальный движок не попадал лишний текст из диалога.
