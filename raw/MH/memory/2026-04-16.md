# Dziennik: 2026-04-16

## Wydarzenia i zmiany z dzisiejszej sesji (Projekt Music AI / Suno / Kie.ai)

### 1. Rozwiązanie problemów z generatorem (Suno API limits)
- **Problem:** API Kie.ai zwracało kod 422: `The length of prompt words in non custom mode cannot exceed 500 characters`. Tekst od DJ Marka przekraczał 700 znaków i trafiał bezpośrednio w tryb `non-custom`.
- **Rozwiązanie:** 
  - Ustawiliśmy `customMode = true` dla API dla zakładki `AI`. W trybie `custom` przekazujemy `prompt` jako czysty tekst piosenki (Lyrics), a `tags` jako styl. 
  - Wymagało to zmiany formatu wyjściowego z DJ Marka, aby dostarczył DWA oddzielne bloki: `---LYRICS---` oraz `---TAGS---` (max 120 znaków). API parsowane jest w `Generator.tsx`.
  - Tryby manualne (Krok po kroku i Samodzielnie) używają `customMode = false` i pomijają całkowicie klucz `tags`, co naprawiło ignorowanie opisu przez Suno. Styl znów zależy w pełni od opisu użytkownika.

### 2. UI / Opcje Generatora
- Usunąłem zdublowany, globalny przycisk "Generuj Hit" na dole strony podczas używania zakładki "AI Kompozytor".
- W oknie wygenerowanego utworu przez AI dodano kafelki z wyborem waluty (1 moneta / 10 not).
- Naprawiono bug "przeskakiwania ekranu" (poprawiony `.scrollIntoView()` na chatbox - dodany atrybut `block: 'nearest'`).

### 3. Logika i Persona AI (DJ Marek)
- **Vibe Muzyczny:** Wdrożono zakaz uzywania stylów rosyjskich w przykładach. Zastąpiono polskimi realiami, z naciskiem na to, co Polacy lubią na imprezach: `Włoskie hity, Disco Polo, Polska Biesiada, Disco 80/90/00 (potężny damski wokal), Rap Uliczny (hooligan)`.
- **Zabezpieczenie przed zawieszeniem:** Jeśli rozmowa z użytkownikiem trwa dłużej niż 10 wiadomości użytkownika, system wstrzykuje nakaz wymuszenia gotowego promptu, Marek mówi *"Dobra, nie ma co gadać, lecimy! Czas to pieniądz!"* i wrzuca tagi.
- **Zakaz auto-promocji:** Dodano zakaz dodawania samego siebie ("DJ Marek ze Szczecina") do tekstu piosenki. Hit ma być w 100% o kliencie.
- **Automatyczne Skróty:** AI samo generuje `---TITLE---`, a w trybach manualnych aplikacja automatycznie uderza w 4 pierwsze słowa z zapytania w celu wygenerowania chwytliwego tytułu.

### 4. Architektura Multi-Agent i Nowe Persony (Line-up)
- **Wdrożenie 3 różnych Agentów:** Przejście z jednego producenta na trójkę artystów (DJ Marek, Maja, Kuba "Kosa").
- **Backend i LLM Routing (OpenRouter):** Punkt `/api/chat-composer` dynamicznie ładuje `System Prompt` bazując na agencie. Dodano mapowanie LLM: Marek -> `google/gemini-3.1-flash-lite-preview`, Maja -> `anthropic/claude-3.7-sonnet`, Kosa -> `x-ai/grok-4-fast`.
- **Dynamiczne Konteksty i Theming UI:** Zmiana agenta (czyszczenie kontekstu pamięci czatu) dynamicznie przełącza kolory (Różowy, Kwasowo-zielony, Pomarańczowy). Dodano przedrostek `ONLINE •` do wszystkich statusów dla pełnej immersji.
- **Avatary UI:** Przebudowano `Generator.tsx` wspierając zdjęcia WebP (`w-full h-full object-cover max-w-none` dla perfekcyjnie zaokrąglonych masek, rozwiązując problem flex-shrink). Wyeliminowano konflikt klas `hidden` vs `flex` stosując natywne style dla fallbackowych ikon.
