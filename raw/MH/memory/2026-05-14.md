## Issues Found & Fixed

### 1. Missing DB Migration (ROOT CAUSE)
- `tracks` table was missing `video_url`, `video_thumbnail_url`, `video_duration_seconds`, `video_expires_at`, `video_task_id` columns
- Migration `add_video_fields_to_tracks.sql` existed but was never applied
- Applied via Supabase MCP ‚Äî all columns now exist
- Backend code that did `UPDATE tracks SET video_url = ...` was silently failing because column didn't exist

### 2. Video Polling Added
- KIE video callbacks appear unreliable (audio callbacks work, video ones don't arrive)
- Added `pollVideoTasks()` function in `index.js` ‚Äî runs every 15 seconds
- Checks `video_tasks` with status `pending`/`processing`, calls KIE `record-info` API
- When video ready ‚Üí updates `video_tasks` + links to `tracks` table
- Located after audio polling, before server start

### 3. Wrong Variant Video (V1 shown for V2)
**Backend ‚Äî `GET /api/tracks/:id`** (index.js ~line 1424):
- Was hardcoding `variant_index = 0` when looking up video in `kie_track_variants`
- Fixed to use `track.variant_index` (the actual variant of the track being loaded)

**Backend ‚Äî `GET /api/video/check`** (index.js ~line 2883):
- Was using `.in('audio_task_id', searchIds)` which could match wrong variant
- Refactored to prioritize search by specific `variant.id`, then fallback to legacy `audio_task_id`

**Frontend ‚Äî `TrackDetail.tsx`**:
- Line 135: Was passing `activeVariant` (always 0) instead of `trackData.variant_index`
