# Архитектура проекта mojhit.pl (Версия 2.0 - Vibe Coding Era)

## 1. Фронтенд (Client App)
- **Стек:** React / Vanilla JS + Tailwind CSS.
- **Компоненты:** Библиотеки готовых блоков (Shadcn/UI, Aceternity UI).
- **Структура:**
  - Hero-секция (траст-бейджи, CTA).
  - 3-шаговый Визард генерации трека (Повод -> Кому -> Детали).
  - Плеер сгенерированных треков (Howler.js / Wavesurfer.js).
- **Хостинг:** Cloudflare Pages или Vercel.

## 2. Бэкенд (Оркестратор)
- **Стек:** Node.js + Express.js.
- **Роуты API:**
  - /api/generate -> Gemini 2.5 Flash (Генерация текста с тегами [Verse], [Chorus]).
  - /api/create-music -> Kie.ai / Suno V5 (Отправка текста на генерацию аудио).
  - /api/check-music/:taskId -> Поллинг статуса трека от Kie.ai.
  - /api/webhooks/stripe -> Прием оплат.
  - /api/webhooks/clerk -> Синхронизация юзеров с БД.
- **Хостинг:** Render, Railway или Fly.io.

## 3. База Данных и Авторизация
- **База Данных:** Supabase (PostgreSQL).
  - Таблицы: Users (баланс кредитов), Tracks (статусы, URL треков), Transactions (оплаты), Settings (API ключи для админки).
- **Авторизация:** Clerk (Современный UI логина/регистрации).
- **Связка:** При регистрации в Clerk срабатывает вебхук, который создает запись в таблице Users в Supabase.

## 4. Хранилище (Storage)
- **Аудио файлы (MP3):** Cloudflare R2 (выбран из-за нулевой стоимости исходящего трафика / Egress = ). Бэкенд скачивает готовый трек от Kie.ai и перекладывает в R2 для вечного хранения.

## 5. Платежи и Безопасность
- **Платежный шлюз:** Stripe (поддержка Przelewy24 / BLIK для Польши).
- **Антифрод (Карты):** Stripe Radar (блокировка подозрительных транзакций).
- **Защита от ботов и DDoS:** Cloudflare на уровне домена.
- **Защита API:** express-rate-limit в Node.js (ограничение генераций с одного IP).
- **Безопасность БД:** Supabase RLS (Row Level Security) - юзер видит только свои треки, баланс меняет только бэкенд.

## 6. Админ-панель (Backoffice)
- **Стек:** Кастомный дашборд на React (закрытый доступ).
- **Функции:**
  - Изменение API-ключей (Stripe, Gemini, Kie.ai) через таблицу Settings в базе (без рестарта сервера).
  - Управление юзерами (выдача кредитов, бан спамеров).
  - Реферальная программа (статистика приглашенных и выплаты).

## 7. Исключенные технологии (Anti-patterns для нашего MVP)
Для максимальной скорости разработки ИИ-агентами (Vibe Coding) мы отказались от:
- Bubble.io (полный уход с no-code платформы).
- Python / Django / FastAPI (выбран Fullstack JS).
- Redis / RabbitMQ / Celery (очереди не нужны, бэкенд просто поллит Kie.ai).
- Elasticsearch (базового поиска Postgres достаточно).
- ELK / Graylog / Prometheus / Grafana (инфраструктурный оверкилл, логи смотрим в консоли хостинга + Sentry для алертов об ошибках).
