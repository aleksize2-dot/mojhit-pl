## [2026-05-25] kie-integration | Open Generative AI — Kie.ai Full Integration

### Context
Шеф нашёл проект Open Generative AI (форк от Anil-matcha, 200+ AI моделей) — опенсорсная студия для генерации изображений/видео/аудио. Его ноутбучный OpenClaw-агент "Claw 🦞" сделал форк с добавлением Kie.ai как провайдера (25-49% дешевле Muapi). Но интеграция была недоделана:
- KieClient был написан но не подключен
- Settings не имел выбора провайдера
- Studio-пакет использовал свой muapi.js без Kie

### Что сделано (5 файлов, +345/-92 строк)

**1. Settings — выбор провайдера** (`components/StandaloneShell.js`)
- Добавлен `<select>` с Muapi.ai / Kie.ai 🦞 в модалке Settings
- Сохраняется в `localStorage.api_provider`

**2. ApiKeyModal** (`components/ApiKeyModal.js`)
- Ссылки на оба провайдера (было только Muapi)

**3. Studio-пакет — главный фикс** (`packages/studio/src/muapi.js`)
- `getProvider()` — чтение `api_provider` из localStorage
- `getBaseUrl()` — Kie: `https://api.kie.ai`, Muapi: через Next.js proxy
- `getAuthHeaders()` — Kie: `Authorization: Bearer`, Muapi: `x-api-key`
- `toKieModel()` — маппинг имён моделей приложения на Kie.ai имена
- `kieSubmitAndPoll()` — трансформация payload в Kie-формат `{model, input: {...}}`
- `kiePollForResult()` — поллинг через `/api/v1/jobs/recordInfo?taskId=`
- `kieEnqueue()` — очередь (макс 2 одновременных запроса к Kie)
- `uploadFile()` для Kie: base64 → `kieai.redpandaai.co/api/file-base64-upload` → реальный URL
- Авто-поля: `quality: basic` для seedream, `reference_image_urls` для seedance

**4. KieClient фиксы** (`src/lib/kieClient.js`)
- Правильные имена моделей (без фейковых google/ префиксов где не надо)
- Только `Authorization: Bearer` (убрал лишний X-API-KEY)
- Только v1 endpoints (v2 не существует)
- `input_prompt` → `prompt`
