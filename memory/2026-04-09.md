# Note: Raw Memory Pipeline

## Summary
The `raw/` directory serves as the **entry point** for a three-stage "Second Brain" architecture (Raw → Compilation → Interface). It functions as a low-friction landing zone for unstructured information—such as documents, screenshots, and logs—which are later processed by an automated LLM pipeline into structured daily logs.

## Key Concepts
*   **Frictionless Ingestion:** Zero-structure requirement allows for immediate capture of diverse media (PDFs, text, logs).
*   **Automated Compilation:** The `compile-raw.js` script automates the transformation of unstructured data into categorized `memory/` entries.
*   **Maintenance Lifecycle:** Raw files are either archived or deleted post-processing to keep the input folder clean.
*   **Knowledge Auditing:** The system includes a `health-check.js` to analyze the contents of the `raw/` folder, ensuring no information silos or untapped domains are ignored.

# Summary: Claude Project Concepts (AI Psychologist & Writer)

## Core Hypothesis
Claude (Anthropic) is the premier model for applications requiring high levels of empathy, nuanced conversation, and sophisticated, human-like text generation, making it ideal for roles like an AI psychologist or creative co-author.

## Key Concepts & Why Claude Excels
1.  **Empathetic & Nuanced Tone:** Claude's "Constitutional AI" training fosters natural politeness, emotional intelligence, and avoids the repetitive, formulaic phrasing often found in other models (e.g., the "ChatGPT flavor").
2.  **Superior Contextual Memory:** Particularly in Sonnet/Opus tiers, it excels at tracking long, complex emotional and narrative threads over extended conversations.
3.  **Application Fit:** These strengths directly translate to two high-potential projects.

## Proposed Projects
### 1. AI Psychologist / Companion
*   **Goal:** Provide a safe, empathetic space for reflection and therapeutic-style dialogue.
*   **Recommended Model:** Claude 3.5/4.5 Sonnet (optimal balance of speed, cost, and depth).
*   **Key Vibe:** Non-judgmental, uses reflective listening and Socratic questioning.
*   **Crucial Consideration:** Requires strict system prompting to establish boundaries and prevent the dispensing of medical advice.
    *   Cross-link: `MEMORY.md#ai-assistant-capabilities` (for guidelines on safe, bounded AI interactions).

### 2. AI Ghostwriter / Co-Author
*   **Goal:** Assist with brainstorming, character/plot development, and generating prose in a specific style.
*   **Recommended Model:** Claude Opus for high-stakes, structural planning and premium prose; Sonnet for faster, iterative brainstorming.
*   **Key Vibe:** Collaborative and adaptable to an author's unique voice.
    *   Cross-link: `MEMORY.md#llm-application-patterns` (for patterns in creative AI collaboration).

## Technical Stack Considerations
*   **LLM Provider:** Kie.ai is noted as a current cost-effective option for accessing Claude models.
*   **System Prompting:** Identified as critically important, especially for defining persona, style, and safety rules for the psychologist application.
    *   Cross-link: `MEMORY.md#prompt-engineering` (for techniques on crafting effective system prompts).

# raw/ – Karpathy‑style memory dump

## Summary
This directory is the **first stage** (Raw) of a three‑stage personal knowledge pipeline. It is designed as an unstructured, low‑friction "inbox" for any form of digital input (articles, notes, screenshots, logs, etc.). The content is later processed by an automated compiler (LLM) to generate structured daily notes in the `memory/` folder, which are eventually curated into the main `MEMORY.md`.

## Key Concepts

1.  **Memory Pipeline:** The system follows a structured flow: `raw/` → **LLM compilation** → `memory/` daily logs → **manual curation** → `MEMORY.md`.
2.  **Raw Inbox:** A deliberately unstructured folder where any digital material can be dumped without prior organization.
3.  **Automated Compilation:** A script (`compile‑raw.js`) uses an LLM to periodically read, summarize, and categorize raw files, transforming them into structured daily notes.
4.  **Health‑Check Integration:** The `raw/` folder is also analyzed during weekly health‑checks to identify untapped or emerging knowledge domains.
5.  **Karpathy Inspiration:** The design is inspired by Andrey Karpathy's "second brain" architecture for personal knowledge management.

## Rules & Workflow
*   **No initial structure** is required.
*   **Optional naming:** `YYYY‑MM‑DD‑description.ext` helps with chronology.
*   **Processing:** Files are processed by an LLM compiler and output to `memory/YYYY‑MM‑DD.md`.
*   **Archive:** Processed files can be moved to `raw/archive/` or deleted.
